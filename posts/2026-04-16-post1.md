# Defining Functional Requirements for a Context-Aware Usability Feedback Community

## Problem Context

Usability testing is an essential part of design practice, but many students lack access to real users and structured evaluation methods. As a result, they often rely on informal peer feedback, which tends to be unstructured, subjective, and difficult to act upon.

## Problem Analysis

The issue is not the absence of feedback, but the absence of context. Without understanding the designer’s intended goal, reviewers often provide irrelevant or superficial comments. For example, feedback on visual style is not useful when the designer is testing navigation clarity. This mismatch reduces the effectiveness of feedback in supporting design iteration.

## Proposed Solution

This project proposes a lightweight community hub that enables rapid usability testing through structured peer feedback. Users are required to define a usability goal when posting their work, and reviewers must respond using guided prompts aligned with that goal.

This approach transforms feedback from free-form commentary into a more focused usability evaluation process.

## Functional Requirements

### Core Requirements

- Users must be able to create posts with a defined usability testing goal.  
- The system must display posts together with their associated goals.  
- Users must provide structured feedback based on prompts such as clarity, confusion, and suggestions.  
- The system must present feedback in relation to the original goal.  

### Optional Features

- Filtering feedback by category  
- Supporting different stages of design  

### Scope Decisions

Advanced features such as expert matching or automated analysis are excluded to maintain a focused and achievable scope.

## Alternative Approaches

Allowing free-form feedback would reduce user effort but likely result in low-quality and misaligned responses. Automated feedback systems could provide structured insights, but introduce technical complexity and reduce the diversity of human perspectives. Therefore, structured peer feedback offers a balanced solution.

## Trade-offs

Structured feedback improves relevance and actionability but reduces flexibility. Requiring users to define goals increases clarity but adds friction. These trade-offs are acceptable, as the system prioritises meaningful insights over casual interaction.

## Technical Alignment

Structured input fields directly support the goal of improving feedback relevance. By constraining responses, the system ensures alignment with the intended usability focus while simplifying data organisation.

## Evaluation and Responsibility

The system can be evaluated by assessing whether feedback is more relevant and actionable compared to unstructured systems. Accessibility considerations include proper labelling, keyboard navigation, and clear hierarchy. The system should also encourage respectful feedback to minimise harm.

## Conclusion

By embedding usability testing principles into a community setting, this system aims to improve the quality and usefulness of peer feedback for design students.
