# Defining Functional Requirements

## Project idea
My web app prototype is about helping university students manage their daily tasks more efficiently.

## Target users
The main users are students who need a simple and clear way to organise their tasks.

## Core functional requirements
1. Users need to create tasks.
2. Users should be able to view and organise tasks.
3. The system must display tasks clearly.

## Optional features
At this stage, I consider features like reminders or sharing optional because they increase complexity but are not essential for the prototype.

## Constraints and trade-offs
Because I am still developing my technical skills, I need to reduce scope and focus on usability rather than advanced features.
