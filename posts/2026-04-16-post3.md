# Designing the Feedback Interaction Flow

## Problem Context

Even with well-defined functional requirements, a system can fail if the interaction flow is unclear or inefficient. In usability-focused platforms, users need to quickly understand how to submit work, review others’ designs, and provide meaningful feedback without unnecessary complexity.

## Problem Analysis

A key challenge is balancing structure and usability. While structured feedback improves quality, overly complex flows can discourage participation. If users are required to navigate too many steps or fill excessive forms, they may abandon the process.

Additionally, the system must support two distinct roles: the designer (who posts work) and the reviewer (who provides feedback). A poorly designed flow may confuse these roles or create unnecessary friction.

## Interaction Design

The proposed interaction flow is designed to be simple and goal-oriented:

1. **Create Post**  
   The designer submits a post including a title, description, and a defined usability testing goal.

2. **Browse Feed**  
   Users view a list of posts, each displaying its usability goal to provide context.

3. **View Post Detail**  
   Users can access a detailed view of a post, including its description and existing feedback.

4. **Provide Feedback**  
   Reviewers submit structured feedback using predefined prompts such as clarity, confusion, and suggestions.

This flow ensures that users can move from discovery to contribution with minimal steps, while maintaining the structured nature of feedback.

## Alternative Approaches

One alternative would be a more complex workflow involving multiple stages, such as separating feedback into different screens or requiring users to complete guided evaluation forms. While this could increase depth, it would also increase friction and reduce usability.

Another approach would be allowing users to comment directly from the feed view. This would improve speed but reduce context, as users may not fully engage with the post before responding.

The chosen flow prioritises clarity and focus over speed and complexity.

## Trade-offs

The simplified interaction flow reduces user effort and encourages participation, but it limits the depth of evaluation that can be performed in a single interaction.

Additionally, separating roles into posting and reviewing clarifies the user journey, but may oversimplify real-world scenarios where users perform both roles simultaneously.

These trade-offs are acceptable, as the system aims to support quick and lightweight usability testing rather than comprehensive evaluation.

## Conclusion

A clear and efficient interaction flow is essential for supporting structured usability feedback. By minimising friction while maintaining guidance, the system enables users to participate easily while still producing meaningful and relevant insights.
