# Defining Functional Requirements

## Project idea
My web app prototype is about helping users manage their daily tasks more efficiently.

## Target users
The main users are university students who need simple tools to organise their tasks.

## Core functional requirements
1. Users need to create tasks.
2. Users should be able to view and organise tasks.
3. The system must display tasks clearly and simply.

## Optional features
At this stage, I consider features like notifications or sharing optional because they increase complexity but are not essential.

## Constraints and trade-offs
Because I am still developing my technical skills, I need to reduce the scope and focus on a simple and clear interface rather than advanced functionality.
